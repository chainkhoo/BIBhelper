# AIAHelper 需求规格说明书

## 1. 引言
### 1.1 编写目的
本说明书用于明确 AIAHelper 的功能、数据、接口与约束，为后续开发、测试、培训与运维提供统一依据。

### 1.2 项目背景
AIAHelper 旨在自动化处理储蓄险、重疾险的 PDF 建议书，生成结构化的 Word/PDF 总结书，减少人工复制粘贴与计算错误。

### 1.3 术语与缩写
| 术语 | 说明 |
| --- | --- |
| 建议书 | 保险公司输出的计划书 PDF，包含保费、现金价值等数据 |
| 总结书 | 面向客户的解读文件，由模板填充生成 |
| 对比总结书 | 同一客户两个储蓄险方案的对照输出 |
| IA 保费征费 | 保险业监管局征收的附加费用 |

## 2. 总体描述
### 2.1 产品定位与范围
- 本地 CLI 工具，不依赖云端服务。
- 负责“文件扫描 → 任务制定 → 数据解析 → 模板填充 → Word/PDF 输出”。
- 仅处理当前目录下的 PDF 文件，输出到同级目录下以客户姓名命名的文件夹。

### 2.2 用户角色
| 角色 | 需求 |
| --- | --- |
| 理财顾问/内勤 | 快速生成总结书、对照多方案、导出 PDF |
| 技术支持 | 维护脚本与模板、扩展险种、排障 |

### 2.3 依赖与假设
- 运行主机具备 Python 3.9+，可安装虚拟环境。
- PDF 来自 AIA 官方模板，结构稳定且包含“建议书摘要”“现金提取举例”等章节。
- 所需 Word 模板均已放置在 `templates/` 或 `老模版/` 中，字段命名与脚本一致。
- 设备可联网以获取实时汇率，若无法联网则接受默认汇率 7.25。

### 2.4 约束
- 文件系统权限可能限制，脚本需具备写入输出目录的权限。
- PDF 提取依赖 `pdfplumber`，若 PDF 被加密需提前解锁。
- macOS 优先，Windows 可兼容但 Pages 转换不可用。

## 3. 功能需求
### F1. 运行准备与依赖管理
- **描述：** 启动脚本时自动检测是否处于虚拟环境，若否则创建 `aiahelper_pro` 并安装 `colorama/pdfplumber/requests/python-docx/docx2pdf`，之后重新执行脚本。
- **输入：** 无，自动执行。
- **输出：** 终端提示依赖状态，若失败则报错退出。
- **验收：** 首次运行可在 3 分钟内完成环境准备，后续运行直接进入业务流程。

### F2. PDF 扫描与分类
- **描述：** 枚举当前目录下的 PDF，提取缴费年期、年龄、姓名，根据规则判断为储蓄险/重疾险；补充文件名关键词判断。
- **输入：** PDF 文件。
- **处理：** `extract_payment_term_and_age` + `classify_by_payment_term_and_age`。
- **输出：** `classified_pdfs` 字典、`FILE_METADATA`。终端显示每个文件的险种、缴费年期、年龄。
- **验收：** 对 90% 以上样本分类正确，无法识别的文件需列入“未识别类型”。

### F3. 任务决策与交互
- **描述：** 系统根据分类结果生成默认任务（单方案/对比），展示执行建议；用户可确认或进入手动模式选择文件/对比对。
- **输入：** `classified_pdfs`、`file_metadata`、用户交互输入。
- **输出：** 待执行任务列表（险种、模式、关联文件）。
- **验收：** 用户输入非法值时给出提示并继续流程；至少能生成一个任务，否则报错退出。

### F4. 数据解析
- **描述：** 针对不同险种执行专用解析函数，提取保费、现金价值、退保金额、倍数等字段。
- **输入：** 每个任务的 PDF 文本、共享元数据、USD/CNY 汇率。
- **处理：**
  - `parse_savings_plan`：匹配“建议书摘要”“现金提取举例”表格，计算 `age_plus_6/withdraw_multiple` 等派生字段。
  - `parse_critical_illness_plan`：读取保障额度、现金价值、退保金额，按索引区分方案 1/2。
- **输出：** 字段字典，含共享数据（姓名、年龄、缴费期）。
- **验收：** 对示例 PDF，关键金额字段与原文一致或误差 <1%；若缺失则记录 WARN 并允许人工补录。

### F5. 模板填充与文件输出
- **描述：** 根据险种选择模板（若缺失则回退到单独模板），将字段替换为格式化值，将 Word/PDF 输出到“客户名/【险种总结书】姓名_年龄_方案（年交X万美金）.docx”。
- **输入：** 模板路径、字段数据、任务信息。
- **输出：** Word 文档，若启用则生成 PDF；终端打印保存路径。
- **验收：**
  - 模板不存在时立即提示并中止当前任务。
  - 生成的文件夹已存在则复用，不覆盖其他客户。
  - 占位符全部替换，无 `{}` 残留。

### F6. PDF 转换管理
- **描述：** 运行前检查 Pages、Word、LibreOffice、textutil，执行多策略转换；全部失败则给出人工建议。
- **输入：** 生成的 docx 路径。
- **输出：** pdf 路径或失败提示。
- **验收：** 至少有一种方法成功时返回 PDF 并移动到客户目录；失败时不影响 Word 文件。

### F7. 汇率获取
- **描述：** 依次请求三组 API，获取 USD/CNY 汇率；若全部失败则使用默认值 7.25，并允许在环境变量中覆盖。
- **输入：** 无显式输入。
- **输出：** 浮点汇率。
- **验收：** 网络异常时最多等待 5 秒/接口，最终一定返回数值。

### F8. 运行日志与错误处理
- **描述：** 所有关键步骤通过彩色日志输出，包括扫描摘要、任务编号、异常栈；KeyboardInterrupt 需优雅退出。
- **验收：** 日志可帮助用户定位失败文件；未捕获异常会打印堆栈并退出码≠0。

## 4. 非功能需求
| 类别 | 需求 |
| --- | --- |
| 性能 | 100 页以内 PDF 的解析需在 10 秒内完成；批量 10 份建议书可在 2 分钟内完成 Word 导出。
| 可用性 | CLI 提示双语（emoji + 中文），关键操作需确认；失败任务不影响其他任务。
| 兼容性 | 支持 macOS 13+ 与 Windows 10+；若缺少某些转换工具应给出安装指引。
| 可维护性 | 解析函数按险种拆分，PLAN_CONFIG 可扩展；模板路径集中在配置中。
| 安全 | 不上传用户数据；仅在本地文件系统产生输出。

## 5. 数据定义
- **文件元数据：** `FILE_METADATA`（绝对路径 → 缴费年期、年龄、姓名、险种标签、是否分段）。
- **任务结构：** `{type, mode, files[]}`。
- **模板字段：** `name`, `age`, `premium_usd`, `cashout_cny_65`, `withdraw_multiple` 等（详见模板/解析函数）。
- **输出命名：** `【{险种}总结书】{姓名}_{年龄}岁_{方案标签}（年交X万美金）.docx/pdf`。

## 6. 外部接口
| 接口 | 方向 | 协议 | 说明 |
| --- | --- | --- | --- |
| Exchangerate API | 出站 | HTTPS GET | 获取 USD/CNY 汇率，超时 5 秒 |
| LibreOffice/Pages/Word | 本地应用 | CLI/AppleScript | 调用以转换 PDF |
| 文件系统 | 读写 | 本地 | 读取 PDF/模板，写入输出目录 |

## 7. 业务流程概述
1. 启动脚本，完成环境检查与 PDF 转换能力检测。
2. 扫描目录 → 分类 → 输出汇总表。
3. 生成任务建议并等待用户确认/调整。
4. 顺序执行任务：
   - 解析 PDF → 组装字段。
   - 创建客户目录 → 生成 Word → 可选转换 PDF。
5. 打印结果摘要（成功/失败数），输出路径供用户分发。

## 8. 验收与测试建议
- **单元测试：** 针对 `_extract_total_premium`、`classify_by_payment_term_and_age`、`create_output_directory_and_save_files` 编写样本测试。
- **集成测试：** 使用至少 3 份储蓄险/重疾险 PDF，验证分类、字段提取、模板输出与储蓄险对比任务。
- **回归测试：** 新增模板或解析逻辑前，构建样本库并对比关键字段。
- **用户验收：** 真实顾问完成 5 份总结书，确认金额、表述、命名均正确。
