# AIAHelper 项目立项文档

## 1. 项目背景
- AIA 代理及理财顾问需要将储蓄险、重疾险的建议书转换为客户可读的总结书，目前主要依赖手工复制粘贴，效率低、错误率高。
- 现有 PDF 中包含多语言文本、嵌套表格以及特殊编码，传统 OCR 难以稳定提取。`aia.py` 通过 `pdfplumber` 和自定义 `ToUnicode` 解析（例如 `_decode_special_sequences`）提升了字符识别准确度。
- 各渠道沿用多套 Word 模板（`templates/` 及 `老模版/`），人工替换字段耗时，且难以及时生成对比方案。

## 2. 项目目标
### 2.1 总体目标
构建一套本地运行的自动化工具，完成“PDF 扫描 → 方案识别 → 数据提取 → 模板填充 → Word/PDF 输出”的闭环，确保总结书 1 分钟内可生成，误差低于 1%。

### 2.2 分阶段目标
1. **V1（当前）：** 支持储蓄险、重疾险的单方案与对比总结书生成，自动建虚拟环境并拉取依赖，按客户姓名创建输出目录。
2. **V2：** 扩展更多险种模板、引入批处理队列、输出进度与失败告警。
3. **V3：** 结合云端 API，提供集中式模板与版本管理，支持审批记录与指标追踪。

## 3. 项目范围
### 3.1 范围内
- 自动扫描当前目录 PDF，依据缴费年期/年龄/文件名判定险种并分组（`scan_and_classify_pdfs`）。
- 生成系统建议任务并允许人工覆写（`determine_tasks` 的交互流程）。
- 针对储蓄险、重疾险解析关键财务指标，填充指定 Word 模板。
- 调用 Pages/Word/LibreOffice/textutil/docx2pdf 多策略将 Word 转换为 PDF，并记录结果。
- 获取美元兑人民币汇率并缓存到当次批处理。

### 3.2 范围外
- 云端部署与 Web 化界面。
- PDF 文件的批量下载、命名规范化及存档策略。
- 模板在线协作、版本审批与权限控制。
- 外部 CRM/投保系统的直接集成（当前仅输出文件）。

## 4. 干系人及职责
- **项目发起人 / 业务主管：** 明确业务边界、优先级、模板需求，提供样本数据。
- **产品负责人：** 维护项目愿景，拆解阶段目标，规划迭代路线图。
- **技术负责人：** 设计解析算法与流程，评估依赖与安全方案。
- **实现工程师：** 编码、测试、封装 CLI，并维护虚拟环境。
- **终端用户（理财顾问、内勤）：** 提供反馈、验证输出准确性。

## 5. 主要交付物与成功标准
| 交付物 | 描述 | 验收标准 |
| --- | --- | --- |
| 自动化脚本 `aia.py` | 完整 CLI 与解析逻辑 | 能在 macOS/Windows 上运行，自动安装依赖并生成结果 |
| 模板资源 | `templates/` 中的标准模板与 `老模版/` 中的兼容文件 | 模板字段与占位符与脚本保持一致，无缺失 |
| 使用文档 | 运行指南、故障排查、模板约定 | 业务团队可独立操作，常见问题可自助解决 |
| 输出文件 | 客户名目录下的 Word/PDF | 字段填充正确，命名规则统一 |

## 6. 里程碑（示例）
1. **M1：需求冻结 & 样本收集（第 1 周）** — 拿到全部模板、典型 PDF。
2. **M2：解析引擎完成（第 3 周）** — 可正确分类并抽取 90% 字段。
3. **M3：模板联调（第 4 周）** — Word/PDF 输出通过业务验收。
4. **M4：试点上线（第 5 周）** — 5 名顾问试用，记录问题。
5. **M5：正式推广（第 6 周）** — 发布版本，进入维护期。

## 7. 资源与技术方案
- **语言与运行环境：** Python 3.9+，脚本自动创建 `aiahelper_pro` 虚拟环境并安装 `colorama/pdfplumber/requests/python-docx/docx2pdf`（`aia.py:20-45`）。
- **核心依赖：** `pdfplumber` + 自定义 `ToUnicode` 映射提升文本解析，`python-docx` 完成模板替换，`docx2pdf`/Pages/LibreOffice 负责 PDF 转换。
- **资产管理：** 模板保存在仓库根目录，输出按客户名建目录并附带参数化文件名（`aia.py:2008-2078`）。
- **外部接口：** 汇率 API 三重备选，失败回落到默认值 7.25（`aia.py:1926-1945`）。

## 8. 风险与应对
| 风险 | 影响 | 缓解措施 |
| --- | --- | --- |
| PDF 结构变动导致解析失败 | 输出字段缺失 | 建立样本回归库，增加 `_extract_total_premium` 等多策略匹配（`aia.py:420-475`） |
| 模板字段与脚本字段不一致 | Word/PDF 占位符残留 | 设置模板校验清单，新增回退机制（`execute_single_task` 模板检测，`aia.py:2210-2234`） |
| 终端缺少 PDF 转换软件 | 无法产出 PDF | 运行前检查 Pages/Word/LibreOffice/textutil，并提示安装建议（`aia.py:2069-2173`、`2314-2345`） |
| 汇率 API 不可用 | 金额换算不准确 | 多个 API 轮询并设置默认值，允许人工覆盖 |
| 操作误差 | 任务执行顺序不当 | 通过 `_print_execution_suggestion` 和交互式确认减少误操作（`aia.py:765-808`） |

## 9. 成本效益预估
- **成本：** 开发 1 人月、模板梳理 0.5 人月、试点培训 0.2 人月。
- **收益：** 每份总结书平均节省 20-30 分钟，按日均 10 份估算，一名顾问每月节省 >40 小时；统一模板降低合规风险，支持快速对比方案提升转化率。
